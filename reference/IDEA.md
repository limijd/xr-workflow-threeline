我最近发现了一个非常好的用AI辅助开发的方法 这个方法叫做三主线战斗流边战边学 核心思路是1让AI训练我使用AI开发不熟悉领域的项目 。 所以这样一个项目都有三条主线 一条是如何用AI就是Harness Engineering，提示词文档项目管理等技巧和规范 ，一条是学习这个项目需要用的知识  另一条主线就是完成项目本身 交付。我主要用的是codex CLI和Claude code CLI 。 

我的要求是这样的

假设开始一个新的需要交付的项目，用户没有领域知识，用户没有AI辅助开发经验

整个三线流战法完全由THREELINE_PLAN.md驱动

使用本skill  threeline-workflow , 这个workflow会初始化项目，第一步是和用户交互初始化文档结构，参见 ~/.claude/CLAUDE.md和w-skill-project-docs, 这一步最好是交互式教学式的一个一个文档在项目文件夹里创建，新创建的文档只需要最小化的初始化内容，每一个只需要和用户确认即可，将来这些文档的内容会不断丰富的。


后面是指导用户通过superpowers:brainstorm和用户对话建立项目认知，目的和初始的function spec. 


然后是design spec

然后是implementation plan, 三线战法的implemeantaion plan可以直接换成用THREELINE_PLAN.md驱动, 所以THREELINE_PLAN.md不仅需要有完整的任务结构，还需要有状态

三线战法的每个实现节点里，都需要包含前备知识的学习，考核，然后开始任务，继续并总结考核本任务知识，结合费曼学习法

只有领域和任务知识考核完成通过，才能进入下一个任务，

整个过程中新引入的Harness Engineering方法需要显示的提醒用户和用户确认，新产生的资料也要汇报给用户。

如果某个节点，客户的知识面不足够，需要提示客户进行系统化学习，这个时候，用户可以选择在项目内学习，也可以选择脱离项目，另开独立的项目学习，如果在项目内学习，AI可以辅助用户收集整理学习资料，安排学习内容，组织学习流程。
